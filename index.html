01 / Meta · About Instagram Meta

IG Web Design System

TPM & program lead — authored the roadmap priority, ran a 5-phase execution across design, engineering, and brand

FigmaDesign Tokens WCAG / AccessibilityComponent Architecture MCDS Alignment

The Problem

About Instagram had no design system on its Consumer website — every build started from scratch, with no shared components, no documented accessibility or SEO standards, and no consistent path from Figma to code, every launch was done on a page-by-page basis with no scalability. Build costs and time compounded with every release, with no consolidated source of truth for Instagram's Consumer Experience design language — and no way to know if a component would even hold up on the next page it touched.

What I Built

Authored and drove the IG Web Design System as a P1 2026 roadmap priority — the first design system for any Instagram web domain. Ran a 5-phase execution: audit and gap analysis against Meta's Centralized Design System (MCDS), module ownership assignment, a multi-stage review (internal, MCDS team, Brand), full documentation (usage guidelines, WCAG accessibility specs, dev handoff notes), and a final team-wide approval gate. Aligned naming conventions, spacing, variants, and accessibility standards to MCDS so the system would scale beyond one team, and framed the business case for reuse across IG Business and IG Creators.

Impact

1st design system shipped for any Instagram web domain
2 surfaces positioned for cross-adoption (IG Business, IG Creators)

03 / Meta · About Instagram Meta

AI-Powered Content Update Automation

TPM & AI systems designer — identified workflow, designed automation, documented outcomes

AI / ClaudeCMS Integration XML TemplatingWorkflow Design Chat Notifications

The Problem

Content updates required manual diff reviews — a team member had to read each change, determine what CMS XML to write, and paste it in. At 15 minutes per task, this was a significant recurring drain with high error potential and no scalability path. The bottleneck often led to outdated content on the marketing website, directly impacting user engagement — one of the team's primary KPIs.

What I Built

An AI-driven automation that detects content changes requiring CMS updates, generates the ready-to-paste XML, and creates a ticket — removing the entire manual review step. I also built a chat notification system that alerts the team the moment an update is available, and a weekly sweep to catch anything that may have been missed. The team goes from receiving a change to merging it in minutes, not quarter-hours.

Impact

85% reduction in task time
15→2 minutes per task to production
04 / Meta · About Instagram Meta

AI-Driven Web Development Planning

TPM — redesigned the kickoff-to-implementation pipeline using AI at every phase

AI / ClaudeFigma Ticket SystemsCreative Brief Eng Planning

The Problem

Web development cycles were long and front-loaded with manual planning work. Design handoffs to engineering required extensive TPM translation, and ticket creation was a time-intensive bottleneck that delayed build starts.

What I Built

An end-to-end AI-assisted planning pipeline: stakeholder meeting → AI-generated creative brief → Figma-to-implementation plan → AI-structured engineering ticket set. Engineers receive a complete task breakdown they can execute by feeding directly into AI tooling — with tasks already fully detailed, implementation time was reduced significantly by eliminating the back-and-forth typically needed to clarify scope and requirements.

p class="block-label">Technical Depth

This pipeline worked reliably because it wasn't just "AI reads a screenshot" — it fed directly on the documented, named component system I'd already built for About Instagram. AI-assisted design-to-code only holds up when components and tokens are mapped rather than inferred from pixels; without that structure, the model has to guess at spacing, color, and hierarchy from scratch every time. That's the same principle behind newer tooling like Figma's own MCP server and Code Connect, which map design components to real code so an agent reuses existing patterns instead of reinventing them. The tradeoff worth naming: business logic and interaction state don't survive design-to-code translation — Figma can't tell you what happens on a button click — so that still required a human-written spec layered on top.

Impact

3mo→5wk web development build time
1wk→4hr planning time per build
05 / Google Ads via R/GA Google · R/GA

SSO Personalization on Google Ads Consumer Site

Technical Lead, 2022 — cross-functional program across product, engineering, legal, and privacy

GAPIFirst-Party Auth Google People APIOAuth 2.0 A/B TestingPrivacy / Legal

The Problem

The Google Ads consumer site served a one-size-fits-all experience, missing a key goal to personalize content, surface product recommendations, and persist user settings via Single Sign-On — with a hard technical constraint: the site ran on a static server with no native first-party cookie support.

What I Led

Designed and shipped the Crawl phase of a three-phase personalization roadmap: an SSO card integration using GAPI, First-Party Authentication, and the Google People API to authenticate users and surface profile data — navigating legal, privacy, a11y, and multi-country/language requirements throughout. All phases gated by A/B testing.
